ARCHITECTURE·技术架构

用什么 搭起来的

个人项目 · 设计 + 全栈开发 · 2026 · 约 3,000 行代码 / 一个长周末

React 18 Vite 5 Inline SVG Web Audio API requestAnimationFrame CSS keyframes localStorage feTurbulence filter
01游戏循环 requestAnimationFrame 驱动 ~60 fps · 角色物理（重力 / 跳跃 / 地形起伏） · 视差摄像机跟随
02手绘风 所有边框过同一个 SVG 滤镜 — feTurbulence + feDisplacementMap — 一行 CSS 全场抖动
03音效 Web Audio API 当场合成 — 脚步、跳跃、水花、星星、开门、通关，8 段过程音效 + 一段循环 BGM
04响应式 手机端去掉键盘改成「前进跳」浮动按钮 · 三种输入（触屏 / 鼠标 / 键盘）共一套状态机
05作品页跳转 iframe 内嵌作品介绍页，边上贴一个迷你小人当返回键 — 可拖动、可切左右
06持久化 localStorage 记住通关次数、跳跃按钮位置、探头小人停靠点

HOW IT WORKS·运作方式

三步 ,一次漫游

从打开网页到通关，只需要做三件事 — 选 · 走 · 看。

01
选一种打开方式

开场卡片两个按钮 — 玩一遍 → 或 直接看作品。
想散步的人开始往右走；急的人直接跳到全部作品页面。

02
走 · 点 · 跳

点屏幕（或按 ← →）走路，路过招牌点开互动 — 敲门聊聊、跳过水坑、点亮灯笼。
路上遇到两件真正的产品，可以当场 ▶ 小演示 或 详细了解 → 跳到完整介绍页。

03
通关 + 重玩

走到终点 ✦ 弹出一张统计卡 — 停靠点 · 星星 · 用时 · 通关次数。
再走一遍，从第二次开始路上会跟着一只小猫 :)

ENGINEERING NOTES·设计取舍

视觉之外的 几个决定

SAME-TAB · 探头小人替代开新标签
点「详细了解」不开新标签 — 把作品介绍页装进 iframe，边上贴一个迷你小人当返回键。可拖动 / 可切左右 / 气泡 5 秒一闪。让「跳出去看完再回来继续走」不打断节奏。
ONE FILTER · 一个滤镜，全场手绘
所有边框、按钮、地形、人物都走同一个 filter: url(#wobble)。改一次 baseFrequency 全站抖动幅度跟着变。
INPUT PARITY · 三种输入一套逻辑
键盘 ← → / 空格、触屏点击、鼠标点击 — 都汇聚到同一个状态机。手机端没有空格键就把跳跃按钮做成可拖动的浮动球。

WHAT'S NEXT·下一版方向

从 「走一段路」 到 「逛一间屋」

朋友试玩后的建议 — 把作品放进一间收纳房间，每件家具都是一个互动点；走路版保留为另一种打开方式。

1
书桌DESK · FOCUS

坐到书桌前打开屏幕 — 启动专注圈 AI 番茄钟。

2
唱片机RECORD · MOODTUNE

走到唱片机旁，告诉它今晚的心情 — AI DJ 压一张唱片。

3
涂鸦墙WALL · MINI EXPERIMENTS

墙上挂着 6 张能点开的小作品 — 字母重力场、纸鹤、夜里的钟……

4
攀岩角BOULDER · MINI GAME

角落一面小攀岩 — 原计划的「攀岩版」收成涂鸦墙里的一个单独小游戏。

两种打开方式 VERSION SWITCH

「走路版」保留为另一种打开方式 — 喜欢漫游的人可以照旧 60 秒走完。
右上角 / 设置里一键在散步 和 房间之间切换。

攀岩版收成小游戏 SCOPE PAUSED

原本想做的「整套纵向攀岩版」复杂度太高 — 暂停。
把它的核心手感留下来，作为单独可玩的 mini-game 放进涂鸦墙里，不再要求走完全程。
